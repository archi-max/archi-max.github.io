  <h1 className="font-display text-4xl md:text-5xl font-semibold text-heading mb-6">Welcome to the Scratch-powered blog</h1>

  <p>
    This blog uses <strong>MDX</strong> via the Scratch toolchain. Posts live in <code>pages/posts</code>, keep their
    own frontmatter (title, date, read time, description, tags, slug), and compile into static pages that the
    portfolio serves under <code>/blog</code>. The typography leans LessWrong: crisp headings, generous line-height, and
    subtly highlighted quotations.
  </p>

  <blockquote>
    Longform writing deserves quiet, respectful styling: calm colors, generous white space, and obvious affordances for
    links and code.
  </blockquote>

  <h2>How to add a post</h2>
  <ol>
    <li>Create a new file in <code>pages/posts/your-title.mdx</code>.</li>
    <li>Include frontmatter: <code>title</code>, <code>date</code>, <code>readTime</code>, <code>description</code>, <code>tags</code>, <code>slug</code>.</li>
    <li>Write Markdown + React components. Use callouts, code blocks, tables—anything MDX supports.</li>
    <li>Run <code>npm run build:blog</code> from the repo root (or <code>npm run build</code> inside <code>blog/</code>).</li>
    <li>The build copies static output to <code>public/blog</code> and emits <code>posts.json</code> for the portfolio cards.</li>
  </ol>

  <h2>Why keep the LessWrong flavor?</h2>
  <p>
    LessWrong’s styling emphasizes clarity: readable fonts, understated borders, and consistent line lengths. That same
    intent shows up here so essays feel cohesive with the rest of the portfolio. We re-used the palette, typography, and
    prose overrides from the main site.
  </p>

  <h2>Embedded code & callouts</h2>
  <p>MDX lets you mix prose and code:</p>

  <pre><code>{`function reliabilityChecklist(system) {
  return [
    'Monitor latency & errors',
    'Prefer idempotent handlers',
    'Design degraded modes',
    'Add backpressure before retries',
  ];
}`}</code></pre>

  <p>
    Want callouts? Just make a component in <code>src/markdown</code> and use it inside your MDX.
  </p>

  <h2>Next steps</h2>
  <ul>
    <li>Add more posts.</li>
    <li>Tweak <code>blog/src/tailwind.css</code> and <code>tailwind.preset.cjs</code> for any aesthetic changes.</li>
    <li>Extend the manifest generator if you want summaries or hero images on the listing page.</li>
  </ul>

  <p className="text-subtle mt-8">Happy writing!</p>
